#
#              ZRadFile - List of radicals and their position for kanji
#
#
# The file was compiled with the help of the original RADKFILE that is:
#       Copyright 2001/2007 Michael Raine, James Breen and the Electronic
#       Dictionary Research & Development Group at Monash University.
#       http://www.csse.monash.edu.au/~jwb/kradinf.html
#
# Even though I used the data in RADKFILE to help process all these kanji, the
# two files are far from identical. This list is based on the classical radical
# listing of kanji, and it not only contains the name for each radical, but
# even groups kanji based on the position of the radicals in them.
#
# Format:
#
#   * Each line starting with # is comment. Do not process these!
#
#   * Each line starting with $ is a radical descriptor. The descriptor line has
#     the following format:
#           $radical_number radical stroke_count name_listing
#     More than one line can have the same number, radical, count etc. The
#     lines with the same radical_number are all different versions of the same
#     radical. They can look slightly different, have different names etc.
#     The name listing can have more than one name for the same radical in the
#     given form and position. These are separated by spaces.
#
#   * The lines following the descriptor contain all kanji for that radical
#     version.
#   
# Trivia for the radical names: (copied from Eyedunno on the Wakan forum)
#   
#   * Hen (偏) are elements on the left side of kanji.
#   * Tsukuri (旁) are elements on the right side of kanji.
#   * kanmuri (冠) or kashira (頭) are elements on the top of kanji.
#   * ashi (脚) are elements on the bottom of kanji.
#   * tare (垂) are elements that cover both the top and left of kanji.
#   * nyou (繞) are elements that cover both the bottom and left of kanji.
#   * kamae (構) are elements that cover two or more sides of kanji, but are not
#     tare or nyou.
#
# Notes on my decision for radical placement:
#   As seen from the trivia above, many radicals were named after their place in
#   the kanji. Some radicals have one (or more) names for their usual form and
#   another one when they are at some given side. Because I had to decide which
#   kanji goes under which of these radicals, I made the following guidelines:
#
#   * When a radical is on the side, top etc. of the kanji and also has a name
#     that matches the place (eg. ends in ...hen or ...kanmuri) that radical
#     is in a separate group but with the same radical number. Only those kanji
#     are listed under this group that have the radical at the given side.
#     Those kanji are not, that have the radical inside the kanji, even if the
#     radical is at the side of some part of the kanji
#
#   * When a radical doesn't have a name for a side where it might be placed, it
#     is in the same group as the main radical.
#
#   * Some radicals are always at the given side (mostly on the top) of kanji
#     parts, they are listed under the same group as the main radical even when
#     they are inside a kanji.
#
#   * Some radicals don't have a name for their main form, only for their
#     place at the sides. These had to be put into separate groups for each side.
#     Because of this, every kanji had to be selected into one of the groups.
#   
#   * No kanji is listed for the main form of the radical, when the radical is
#     placed twice or more times in the kanji and one is on a named side. (There
#     are exceptions when I was lazy, so don't count on this!)
#
#   * Kanji like 林 that are repetitions of parts are always listed under the
#     main radical.
#
# Disclaimer: This list is as is. It might contain errors, so it's up to you how
#             you use it. Some kanji might be listed for the wrong radical or
#             wrong position.
#             I selected the position for each kanji according to how it looked
#             right, but this is not a scientific work of any sort!
#             
# You can use, modify, sell or do whatever you want with this file if you don't
# claim that it is your own! (Read the licence of RADKFILE just in case, though
# I'm not half sure it complies here.)
#
